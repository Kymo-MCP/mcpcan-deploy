Quick Start

📋 Prerequisites: Environment Dependencies Installation Guide

🚀 Helm Repository Installation (Recommended)

# Step 1: Add MCPCan Helm repository
helm repo add mcpcan https://kymo-mcp.github.io/mcpcan-deploy/
helm repo update

# Step 2: Install MCPCan
helm install mcpcan mcpcan/mcpcan-deploy --namespace mcpcan --create-namespace --timeout 600s --wait

🔧 Custom Configuration (Optional)

# Install with custom domain
helm install mcpcan mcpcan/mcpcan-deploy \
  --set global.domain=your-domain.com \
  --set ingress.tls.enabled=true \
  --namespace mcpcan --create-namespace --timeout 600s --wait

# Install with custom resource limits
helm install mcpcan mcpcan/mcpcan-deploy \
  --set services.gateway.resources.limits.memory=512Mi \
  --set services.gateway.resources.limits.cpu=500m \
  --namespace mcpcan --create-namespace --timeout 600s --wait

✅ Verify Installation

# Check deployment status
kubectl get pods -n mcpcan
kubectl get svc -n mcpcan
kubectl get ingress -n mcpcan

# Access the application
echo "Visit: http://$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[0].address}')"

# For domain-based access
echo "Visit: https://your-domain.com"

💡 Need more control? See Advanced Installation & Management for Git clone-based installation and advanced configuration options.

Advanced Installation & Management

📦 Git Clone Installation (Local Development)

# Clone the repository
git clone https://github.com/Kymo-MCP/mcpcan-deploy.git
cd mcpcan-deploy

# Install environment dependencies
bash scripts/install-run-environment.sh

# Basic installation with local Helm chart
helm install mcpcan ./helm --namespace mcpcan --create-namespace --timeout 600s --wait

# Installation with custom values file
cp helm/values.yaml helm/values-custom.yaml
# Edit helm/values-custom.yaml to customize configuration
helm install mcpcan ./helm -f helm/values-custom.yaml \
  --namespace mcpcan --create-namespace --timeout 600s --wait

🔧 Advanced Configuration Options

# Custom image version
helm upgrade --install mcpcan ./helm \
  --set global.version=v1.2.3 \
  --namespace mcpcan --timeout 600s --wait

# Custom domain and TLS
helm upgrade --install mcpcan ./helm \
  --set global.domain=your-domain.com \
  --set ingress.tls.enabled=true \
  --namespace mcpcan --timeout 600s --wait

# Custom resource limits
helm upgrade --install mcpcan ./helm \
  --set services.gateway.resources.limits.memory=512Mi \
  --set services.gateway.resources.limits.cpu=500m \
  --namespace mcpcan --timeout 600s --wait

# Disable specific services
helm upgrade --install mcpcan ./helm \
  --set services.market.enabled=false \
  --namespace mcpcan --timeout 600s --wait

🌍 Multi-Environment Deployment

# Development environment
helm install mcpcan-dev ./helm -f helm/values-dev.yaml \
  --namespace mcpcan-dev --create-namespace --timeout 600s --wait

# Staging environment
helm install mcpcan-staging ./helm -f helm/values-staging.yaml \
  --namespace mcpcan-staging --create-namespace --timeout 600s --wait

# Production environment
helm install mcpcan-prod ./helm -f helm/values-prod.yaml \
  --namespace mcpcan-prod --create-namespace --timeout 600s --wait

🛠️ Deployment Management

# Upgrade deployment
helm upgrade mcpcan ./helm -f helm/values-custom.yaml \
  --set global.version=v1.1.0 \
  --namespace mcpcan --timeout 600s --wait

# View upgrade history
helm history mcpcan --namespace mcpcan

# Rollback to previous version
helm rollback mcpcan 1 --namespace mcpcan

# Uninstall deployment
helm uninstall mcpcan --namespace mcpcan

📊 Monitoring & Troubleshooting

# View Helm Release status
helm status mcpcan --namespace mcpcan

# View Pod status
kubectl get pods -n mcpcan

# View service status
kubectl get svc -n mcpcan

# View Ingress status
kubectl get ingress -n mcpcan

# View specific service logs
kubectl logs -n mcpcan -l app=mcp-gateway
kubectl logs -n mcpcan -l app=mcp-authz
kubectl logs -n mcpcan -l app=mcp-market
kubectl logs -n mcpcan -l app=mcp-web

# View logs in real-time
kubectl logs -n mcpcan -l app=mcp-gateway -f

# View Pod detailed information
kubectl describe pod <pod-name> -n mcpcan

# View events
kubectl get events -n mcpcan --sort-by='.lastTimestamp'

# Port forwarding (local debugging)
kubectl port-forward svc/mcp-gateway-svc 8080:8080 -n mcpcan
kubectl port-forward svc/mcp-web-svc 3000:3000 -n mcpcan

🔒 TLS Certificate Configuration

# Generate self-signed certificate
./scripts/generate-simple-cert.sh demo.mcpcan.com 365

# Certificate files will be generated in certs/ directory
ls certs/
# tls.crt - Certificate file
# tls.key - Private key file

# Deploy with TLS configuration
helm install mcpcan ./helm -f helm/values-custom.yaml \
  --set ingress.tls.enabled=true \
  --namespace mcpcan --create-namespace --timeout 600s --wait

Environment Dependencies Reference

📋 Note: Environment dependencies are automatically installed in both Quick Start and Advanced Installation methods above.

📦 Git Clone Repository

# Clone the repository and run installation script
git clone https://github.com/Kymo-MCP/mcpcan-deploy.git
cd mcpcan-deploy
bash scripts/install-run-environment.sh

📋 Script Parameters

Parameter Description
--cn Use China mirror sources for all components
--force Force reinstall all components
--skip-k3s Skip K3s installation
--skip-helm Skip Helm installation
--skip-ingress Skip Ingress-Nginx installation
--token <token> K3s cluster token
--version <version> K3s version to install
-h, --help Show help information

💡 Tip: After successful installation, return to Quick Start or Advanced Installation to deploy MCPCan.

System Components

Infrastructure Services

🗄️ MySQL 8.0

Primary database for application data, providing persistent storage

🔄 Redis 6

Cache and session storage to improve system performance

Application Services

🌐 Web Frontend (Port: 3000)

React-based user interface providing intuitive operation experience

🏪 Market Service (Port: 8080)

Package management and marketplace functionality, supporting package publishing and downloading

🔐 Authorization Service (Port: 8081)

User authentication and authorization management to ensure system security

🚪 Gateway Service (Port: 8082)

API gateway and routing management, unified service entry point

🔧 Initialization Service (Port: 8083)

System initialization and configuration management, automated deployment process

Supporting Components

📋 ConfigMaps

Configuration management with support for dynamic configuration updates

🔒 Secrets

Sensitive data management, secure storage of keys and certificates

🌍 Ingress

External access routing with domain and TLS support

💾 Persistent Volumes

Data persistence storage to ensure data security
